# Clean Code - Summary

## Book Info
- Title: Clean Code  
- Author: Robert C. Martin (Uncle Bob)  

---

## Chapter 2: Meaningful Names
In this chapter, I learned that choosing good names for variables and functions is very important. A clear name makes the code easier to understand without needing too many comments. Bad names can make the code confusing even if it works correctly.

---

## Chapter 3: Functions
This chapter explains that functions should be small and do only one thing. When a function is too big, it becomes hard to read and maintain. Breaking functions into smaller parts makes the code cleaner and easier to reuse.

---

## Chapter 4: Comments
I learned that comments should not be used to fix bad code. Instead of adding many comments, it’s better to write clear code. Comments are useful only when something needs extra explanation.

---

## Chapter 6: Objects and Data Structures
This chapter explains the difference between objects and data structures. Objects hide data and use methods, while data structures expose data. Understanding this helps in writing better organized and structured code.

---

## Conclusion
This reading helped me understand how to write cleaner and more readable code. I learned that good naming, small functions, and clear structure are very important in programming.
